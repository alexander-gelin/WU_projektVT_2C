1. Idébeskrivning

Jag planerar att skapa en webbsida för ett band som jag repar med varje vecka. Jag tänker att webbsidan ska främst innehålla information om bandet (musiken vi spelar, hur vi väljer låtar, tider, plats osv.). Bandet är en del av Göteborgs Kulturskola och det kan därför även finnas information om man är intresserad att gå med. Målgruppen blir därför musik-interesserade tonåringar/ungdomar som spelar något instrument.

Komplettering

Jag fick några problem som behövde lösas för min komplettering. Den första var att när man minimiserar indexsidan så overlappar layouten med varandra. Dessutom blir slide-bilderna mindre och mindre. Det andra problemet var att footern på de övriga sidorna hade ett problem då den fastnade mitt på sidan. Jag kan börja med att beskriva footer-problemet. Det jag först försökte experimentera var border-bottom/bottom propertyn. Det jag märkte var att bottom propertyn funkar bra på sidor utan någon overflow medan border-bottom inte gör det (se exempel nedan). Så jag experimenterade lite sida för sida för att hitta den optimala propertyn. Alla förutom latar.html (som hade overflow property) fick bottom medan latar.html fick border-bottom. Det enda problem som kvarstår är om man scrollar helt och hållet ner på sidan, minimiserar fönstret och scrollar ner. Jag hade svårt med att försöka fixa det här men den fixar sig själv så fort man förstorar fönstret sen. Angående problemet med overlappande layout på startsidan så använde jag mig av overflow properties och flex-properties. Innan hade mina slideimages en object-fit contain property vilket gjorde att den automatiskt ändrade på sig beroende på sectionens storlek. Nu använder jag mig istället för konstanta pixel värden, flex för att göra så den är positionerad ordentligt i mitten och overflow: hidden så att inte bilderna går utanför section då sidan blir mindre. Det finns säkert en bättre lösning men med tanke på att grid layouten gör så att allt blir i princip bundet till varandra, är detta det bästa jag kom på. Sedan när det kommer till knapparna på startsidan så satte jag där också en overflow property för att dem inte ska gå över på någon annan del av sidan. Som sagt, det bästa jag kom på med tanke på omständigheterna. (bild finns nedan)

Allt som allt var detta kompletteringsarbete ett sätt att hitta lösning till några enstaka problem under kort tid. Jag var lite orolig med att jag inte skulle kunna lösa det (vet ej om jag gjort det fortfarande) men jag tänker att detta fungerar som nuvarande lösningar. Såklart kan man ta det ännu längre om man gör om flera saker i koden för att det ska bli en optimal lösning men jag ville inte riskera att jag måste fixa väldigt många olika saker för att få det att funka. Men med tanke på testresultaten (t.ex. validator) så känner jag ändå att jag blev nöjd med webbsidan som helhet då det ändå är något jag skapat med mina egna kunskaper och justerat med egna erfarenheter.
